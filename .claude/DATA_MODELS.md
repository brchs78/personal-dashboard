# Data Models — JSON Schemas for OLE OS

All persistent data is stored as JSON files in `userData/`. This document defines the exact structure of each file for reference.

---

## 1. training-plan.json

**Purpose:** Marathon training plan state, stores the current weekly/monthly plan with day-by-day breakdown.

**Schema:**

```json
    "totalWeeks": 16,
    "startDate": "2026-05-01",
    "endDate": "2026-08-15",
    "raceDate": "2026-08-16",
    "days": [
      {
        "date": "2026-05-25",
        "dayLabel": "Mo",
        "type": "Easy",
        "title": "Morning Run",
        "distanceKm": 8,
        "durationMin": 48,
        "paceTarget": "5:50-6:00/km",
        "hrZone": "Z2",
        "notes": "Recovered from long run",
        "done": false
      },
      {
        "date": "2026-05-26",
        "dayLabel": "Tu",
        "type": "Tempo",
        "title": "Tempo Session",
        "distanceKm": 10,
        "durationMin": 55,
        "paceTarget": "5:10-5:20/km",
        "hrZone": "Z3",
        "notes": null,
        "done": false
      },
      {
        "date": "2026-05-27",
        "dayLabel": "We",
        "type": "Cross",

```

**Fields:**
- `current.phase`: Training phase name (e.g., "Base Building", "Aerobic Build", "Peak", "Taper")
- `current.weekNumber`: Week within this phase (1-16)
- `current.days[]`: Array of daily training prescriptions
  - `date`: ISO date string (YYYY-MM-DD)
  - `dayLabel`: Day of week abbreviation (Mo-Su)
  - `type`: "Easy", "Tempo", "Threshold", "Intervals", "Long", "Recovery", "Cross", "Yoga+Easy", "Gym+Easy", "Rest"
  - `title`: Human-readable workout name
  - `distanceKm`: Target distance (null if not applicable)
  - `durationMin`: Target duration in minutes
  - `paceTarget`: Pace range or single value (e.g., "5:30-5:40/km")
  - `hrZone`: Heart rate zone ("Z1" - "Z5", or null)
  - `notes`: User/coach notes (null if none)
  - `done`: Boolean marking if day is completed
- `history[]`: Array of previous weeks (for archive/analysis)

## 2. health-data.json

**Purpose:** HealthKit-synced health metrics (sleep, heart rate, steps) with differential update support.

**Path:** `userData/health-data.json`

**Status:** Currently not used; will be generated by `health-sync.js` in Phase 2.

**Schema:**
```json
{
  "sleep": [
    {
      "date": "2026-06-01",
      "duration": 7.5,
      "quality": "good",
      "efficiency": 0.92
    },
    {
      "date": "2026-06-02",
      "duration": 6.2,
      "quality": "fair",
      "efficiency": 0.85
    }
  ],
  "heartRate": [
    {
      "date": "2026-06-01",
      "restingHR": 58,
      "avgHR": null,
      "maxHR": null,
      "hrv": null
    },
    {
      "date": "2026-06-02",
      "restingHR": 59,
      "avgHR": null,
      "maxHR": null,
      "hrv": null
    }
  ],
  "steps": [
    {
      "date": "2026-06-01",
      "count": 12500
    },
    {
      "date": "2026-06-02",
      "count": 15200
    }
  ],
  "workouts": [
    {
      "date": "2026-06-01",
      "type": "Running",
      "distance": 10.2,
      "duration": 3600,
      "pace": "5:52/km",
      "avgHR": 148,
      "source": "export"
    }
  ],
  "lastSync": "2026-06-02T10:30:00Z",
  "lastSyncStatus": "success",
  "lastError": null
}
```

**Fields:**
- `sleep[]`: Daily sleep records
  - `date`: ISO date string
  - `duration`: Hours slept (decimal)
  - `quality`: "unknown", "poor", "fair", "good", "excellent"
  - `efficiency`: 0.0-1.0 (time asleep / time in bed)
- `heartRate[]`: Daily HR summary
  - `date`: ISO date string
  - `restingHR`: BPM (taken in morning)
  - `avgHR`, `maxHR`, `hrv`: Optional; null if not available
- `steps[]`: Daily step counts
  - `date`: ISO date string
  - `count`: Total steps for day
- `workouts[]`: Structured workout records (Phase 3+)
  - `date`: ISO date string
  - `type`: "Running", "Cycling", "Swimming", etc.
  - `distance`: Distance in km
  - `duration`: Duration in seconds
  - `pace`: Pace string (e.g., "5:52/km")
  - `avgHR`: Average heart rate during workout
  - `source`: "export" or "healthkit"
- `lastSync`: ISO timestamp of last successful sync
- `lastSyncStatus`: "success" or "error"
- `lastError`: Error message if lastSync failed, otherwise null

## 4. calendar-cache.json

**Purpose:** Cached calendar events from internal, subscription (iCal), and CalDAV (iCloud) sources.

**Path:** `userData/calendar-cache.json`

**Schema:**
```json
{
  "events": [
    {
      "id": "uuid-internal-001",
      "uid": "ole-internal-001@oleapp",
      "title": "Coach Session",
      "description": "Running form analysis",
      "location": "Central Park",
      "start": "2026-06-05T10:00:00Z",
      "end": "2026-06-05T11:00:00Z",
      "allDay": false,
      "source": "internal",
      "writable": true,
      "sourceLabel": "OLE OS",
      "caldav": null
    },
    {
      "id": "uuid-caldav-001",
      "uid": "UUID@icloud.com",
      "title": "Dentist Appointment",
      "description": null,
      "location": "Downtown Clinic",
      "start": "2026-06-10T14:30:00Z",
      "end": "2026-06-10T15:30:00Z",
      "allDay": false,
      "source": "caldav",
      "writable": true,
      "sourceLabel": "iCloud Calendar",
      "caldav": {
        "calendarUrl": "https://caldav.icloud.com/calendars/p/ole%40icloud.com/calendar/",
        "objectUrl": "https://caldav.icloud.com/calendars/p/ole%40icloud.com/calendar/events/...",
        "etag": "\"abc123def456\""
      }
    },
    {
      "id": "uuid-subscription-001",
      "uid": "event-from-subscription@external.com",
      "title": "Public Holiday",
      "description": "Memorial Day",
      "location": null,
      "start": "2026-05-25T00:00:00Z",
      "end": "2026-05-26T00:00:00Z",
      "allDay": true,
      "source": "subscription",
      "writable": false,
      "sourceLabel": "Holidays",
      "caldav": null
    }
  ],
  "lastSync": "2026-06-02T10:00:00Z"
}
```

**Event Fields:**
- `id`: Unique identifier (UUID or string) for this event in OLE OS
- `uid`: Universal ID (from iCalendar UID property)
- `title`: Event name
- `description`: Event details (markdown allowed)
- `location`: Location string
- `start`: ISO 8601 timestamp
- `end`: ISO 8601 timestamp
- `allDay`: Boolean (if true, start/end are dates only, times ignored)
- `source`: "internal", "caldav", or "subscription"
- `writable`: Boolean (true if user can edit/delete)
- `sourceLabel`: Human-readable source name (e.g., "iCloud Calendar", "Holidays")
- `caldav`: Object (only if source === "caldav")
  - `calendarUrl`: CalDAV calendar collection URL
  - `objectUrl`: CalDAV event resource URL
  - `etag`: ETag for conflict detection on updates

## 5. calendar-caldav-account.json

**Purpose:** Credentials and metadata for iCloud CalDAV connection (one account per installation).

**Path:** `userData/calendar-caldav-account.json`

**Status:** Created on first CalDAV connection, updated on sync.

**Schema:**
```json
{
  "appleId": "ole@icloud.com",
  "passwordEnc": "AgCr/uZ+abc...base64 encrypted password...xyz==",
  "connected": true,
  "selectedCalendars": [
    "https://caldav.icloud.com/calendars/p/ole%40icloud.com/calendar/"
  ],
  "targetCalendarUrl": "https://caldav.icloud.com/calendars/p/ole%40icloud.com/calendar/",
  "lastSync": "2026-06-02T10:00:00Z",
  "lastError": null
}
```

**Fields:**
- `appleId`: Apple ID email
- `passwordEnc`: App-specific password encrypted via `safeStorage.encryptString()`, stored as base64
- `connected`: Boolean (true if credentials valid and last sync succeeded)
- `selectedCalendars[]`: Array of calendar URLs to display/sync
- `targetCalendarUrl`: Calendar URL where coach/user creates new events
- `lastSync`: ISO timestamp of last successful sync
- `lastError`: Error message if last sync failed, otherwise null

**⚠️ Security Note:** Password is NEVER returned to renderer, NEVER logged, NEVER in plaintext files.

## 6. strava-cache.json

**Purpose:** Cache of recent Strava activities (read-only from API).

**Path:** `userData/strava-cache.json`

**Status:** Used; created on first Strava fetch, updated periodically.

**Schema:**
```json
{
  "activities": [
    {
      "id": 12345678901234,
      "name": "Morning 10K",
      "type": "Run",
      "distance": 10.2,
      "moving_time": 3600,
      "elapsed_time": 3800,
      "average_speed": 10.2,
      "max_speed": 12.1,
      "elevation_gain": 120,
      "start_date": "2026-06-02T06:00:00Z",
      "start_date_local": "2026-06-02T08:00:00",
      "achievement_count": 2,
      "kudos_count": 5,
      "comment_count": 0,
      "pr_count": 1
    }
  ],
  "lastFetch": "2026-06-02T10:30:00Z",
  "lastError": null
}
```

**Fields:**
- `activities[]`: Array of recent activities from Strava
  - `id`: Strava activity ID
  - `name`: Activity name (user-given)
  - `type`: "Run", "Ride", "Swim", etc.
  - `distance`: Distance in km
  - `moving_time`: Moving time in seconds
  - `elapsed_time`: Total elapsed time (including pauses) in seconds
  - `average_speed`: Average speed in km/h
  - `max_speed`: Maximum speed in km/h
  - `elevation_gain`: Elevation gain in meters
  - `start_date`: ISO timestamp (UTC)
  - `start_date_local`: Local timestamp
  - `achievement_count`, `kudos_count`, etc.: Metadata
- `lastFetch`: ISO timestamp of last API fetch
- `lastError`: Error message if fetch failed, otherwise null

## 7. todos.json

**Purpose:** Task management with categories, priorities, due dates.

**Path:** `userData/todos.json`

**Schema:**
```json
{
  "todos": [
    {
      "id": "uuid-1",
      "title": "Buy new shoes",
      "category": "gear",
      "priority": 2,
      "dueDate": "2026-06-10",
      "done": false,
      "createdAt": "2026-05-20T10:00:00Z",
      "notes": "Nike Pegasus 40"
    },
    {
      "id": "uuid-2",
      "title": "Physiotherapy session",
      "category": "health",
      "priority": 3,
      "dueDate": "2026-06-05",
      "done": true,
      "createdAt": "2026-05-15T14:00:00Z",
      "notes": "Knee strengthening exercises"
    }
  ]

```

**Fields:**
- `id`: Unique ID (UUID)
- `title`: Task description
- `category`: Category string (user-defined, e.g., "gear", "health", "nutrition", "travel")

- `dueDate`: ISO date string (YYYY-MM-DD)
- `done`: Boolean
- `createdAt`: ISO timestamp
- `notes`: Optional notes

## Data Cleanup & Archiving

- **Old training plans:** Keep in `training-plan.json.bak` (monthly archive)
- **Old events:** Archive to `calendar-archive.json` after 2 years
- **Old todos:** Move completed todos to `todos-archive.json` annually
- **Cache rotation:** Keep last 100 events, 500 activities, 365 days of health data

## Validation & Type Safety

For critical operations (e.g., plan updates, event syncs), validate incoming data:

```js
// Example: Validate day update
function validateDayPatch(patch) {
  const allowed = ['type', 'title', 'distanceKm', 'durationMin', 'paceTarget', 'hrZone', 'notes'];
  const types = {
    type: str => ['Easy', 'Tempo', 'Long', 'Rest'].includes(str),
    title: str => typeof str === 'string' && str.length > 0,
    distanceKm: num => num === null || (typeof num === 'number' && num > 0),
    durationMin: num => num === null || (typeof num === 'number' && num > 0),
    paceTarget: str => str === null || /^\d:\d{2}/.test(str),
    hrZone: str => str === null || /^Z[1-5]$/.test(str),
    notes: str => str === null || typeof str === 'string'
  };
  for (const [key, value] of Object.entries(patch)) {
    if (!allowed.includes(key)) throw new Error(`Invalid field: ${key}`);
    if (!types[key](value)) throw new Error(`Invalid value for ${key}: ${value}`);
  }
  return true;
}
```

Use this for IPC handlers to prevent data corruption.
